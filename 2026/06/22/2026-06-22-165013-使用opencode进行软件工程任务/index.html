一、opencode 简介

opencode 是一款面向软件工程任务的交互式 CLI 工具，它通过集成多种工具和代理系统，帮助开发者高效完成代码搜索、编辑、重构、测试等日常开发任务。opencode 支持 TypeScript、Python、Rust、Go 等多种主流语言项目，并可胜任从微小的代码改动到复杂的跨文件重构等多种场景。

1.1 核心能力

能力 说明 适用场景
代码搜索 基于 grep/glob 的快速内容与文件查找 快速定位函数、类、配置项
文件编辑 精确的字符串替换与文件创建 重构、修复 Bug、添加功能
任务规划 多步骤任务分解与并行执行 复杂功能实现、系统迁移
代理系统 专用代理处理特定类型任务 代码探索、多步骤复杂任务
终端执行 在项目环境中实时运行命令 安装依赖、运行测试、构建

三、核心功能详解

3.1 文件导航与搜索

opencode 提供了三种文件操作工具：

  • Read：读取文件内容，支持指定行范围
  • Glob：按文件名模式匹配，如 **/*.ts
  • Grep：按内容正则搜索，如搜索所有包含 TODO 的行

3.2 文件编辑与创建

  • Edit：精确的字符串替换，支持匹配上下文防误改
  • Write：创建新文件或完全覆写已有文件
  • replaceAll：批量替换所有匹配项（适用于变量重命名等场景）

编辑原理：基于精确字符串匹配而非行号，这使得即使文件后续发生变化，也能正确识别修改位置。

四、实战工作流程

4.1 修复 Bug

  1. 复现与定位：通过搜索相关代码找到 Bug 所在
  2. 理解上下文：读取周围代码理解逻辑
  3. 制定方案：规划修复步骤
  4. 执行修改：使用 Edit 工具修改代码
  5. 验证：运行测试确认修复

4.2 添加新功能

  1. 探索现有实现：了解已有的同类功能模式
  2. 参考约定：查看相邻文件的代码风格和引入的库
  3. 并行实施：同时创建/修改多个相关文件
  4. 检查质量：运行 lint 和 typecheck

4.3 代码重构

  • 使用 replaceAll 批量重命名变量或函数
  • 利用代理系统分析代码结构，制定重构策略
  • 逐步验证，确保每个中间状态都可运行

五、效率技巧

5.1 并行工具调用

opencode 支持在单次响应中调用多个独立工具，大幅提升效率。例如，同时搜索多个模式、读取多个文件。

5.2 善用技能

opencode 的技能系统预置了多种专业能力：

  • frontend-design：创建高质量前端界面
  • docx/pptx/xlsx：办公文档处理
  • pdf：PDF 操作
  • algorithmic-art：生成艺术
  • theme-factory：主题定制

通过 load_skill 机制可以按需加载。

5.3 遵循项目约定

  • 修改代码前先理解现有代码的约定（缩进、命名、库的选择等）
  • 不随意引入新的依赖，尽量使用项目中已有的库
  • 安全最佳实践：不暴露或提交密钥

5.4 使用 AGENTS.md

在项目中维护 AGENTS.md，记录项目特有的规则、工作流程和常用命令，opencode 会在每次会话中自动参考这些规则，保持行为一致性。

六、注意事项

  • URL 安全：opencode 不会生成或猜测 URL，除非确定其用途
  • 隐私保护：密钥和敏感信息不会出现在输出中
  • 代码注释：除非用户要求，opencode 不会添加多余的注释
  • git 操作：只会在用户明确要求时执行 commit 或 push

七、总结

opencode 通过精心设计的工具链和代理系统，将 AI 能力深度融入软件工程工作流。无论是日常的代码搜索、文件编辑，还是复杂的功能实现、系统重构，opencode 都能提供高效、精确的辅助。结合项目特有的配置和技能系统，它能够适应各种技术栈和团队的开发习惯，是现代开发者值得尝试的 AI 编程助手。
