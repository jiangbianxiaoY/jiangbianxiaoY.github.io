## 一、Claude Code 概述

### 1.1 什么是 Claude Code

Claude Code 是 Anthropic 官方推出的 AI 编程代理工具。它运行在你的终端中，能够理解整个代码库，编辑文件、执行命令、管理 Git 工作流，并与你的开发工具集成——全部通过自然语言指令完成。

经过 2025-2026 年的密集迭代，Claude Code 已成为覆盖终端 CLI、VS Code、JetBrains IDE、桌面应用、Web 浏览器、Slack 和 CI/CD 的全方位 AI 编码平台。截至 2026 年 6 月，其 GitHub 仓库已获得超过 **134,000** 个 Star。

### 1.2 核心特性

| 特性 | 说明 |
|:----|:----|
| **代理循环（Agentic Loop）** | 自主完成"收集上下文→执行操作→验证结果"的闭环 |
| **多模型支持** | Claude Opus 4.8 / Sonnet 4.6 / Haiku 4.5 等，支持按需切换 |
| **多界面运行** | 终端、VS Code、JetBrains、桌面 App、Web、Slack、CI/CD |
| **CLAUDE.md 记忆系统** | 持久化项目指令和自动记忆偏好 |
| **MCP 协议集成** | 连接 300+ 外部服务和工具的开放标准 |
| **子代理（Subagent）** | 并行执行子任务，隔离上下文 |
| **钩子系统（Hooks）** | 在编辑、命令等事件前后自动运行脚本 |
| **技能系统（Skills）** | 可共享的可复用工作流 |
| **安全机制** | 快照回滚、权限分级、沙箱执行 |
| **多模态** | 支持图片输入，理解 UI 截图、设计稿 |
| **云执行** | 在 Anthropic 云 VM 中运行，无需本地环境 |
| **远程控制** | 从手机/浏览器继续本地会话 |

### 1.3 可用模型

| 模型 | 代号 | 特点 | 适合场景 |
|:----|:----|:----|:--------|
| Claude Opus 4.8 | `opus` | 最强推理能力 | 复杂架构设计、深度分析 |
| Claude Sonnet 4.6 | `sonnet` | 速度与质量平衡 | 日常开发、大多数编码任务 |
| Claude Haiku 4.5 | `haiku` | 极速响应 | 简单任务、快速迭代 |
| Claude Fable 5 | `fable` | 新模型 | 创意类和创新性编码 |

在会话中通过 `/model` 命令随时切换。

### 1.4 所有可用界面

| 界面 | 启动方式 | 适合场景 |
|:----|:--------|:--------|
| **终端 CLI** | `claude` | 日常全功能开发 |
| **VS Code 扩展** | 插件市场安装 | 内联 diff、@提及、代码内交互 |
| **JetBrains 插件** | 插件市场安装 | IntelliJ/PyCharm/WebStorm 等 |
| **桌面 App** | 独立应用 | 多会话并排、可视化 diff、云会话 |
| **Web** | claude.ai/code | 云端运行，无需本地 |
| **Slack** | @Claude | 从聊天直接委派任务 |
| **GitHub Actions** | CI 配置 | 自动代码审查、PR 创建 |
| **GitLab CI/CD** | 流水线集成 | 自动化流水线 |

## 二、安装指南

### 2.1 系统要求

- **操作系统**：macOS 12+、Linux（Ubuntu 20+）、Windows 10+（推荐 WSL2）
- **终端**：建议使用现代终端（iTerm2、WezTerm、Windows Terminal、Alacritty）
- **Node.js**：使用 npm 安装时需要 Node.js 18+
- **Git**：推荐安装 Git for Windows（若使用原生 Windows 环境）

### 2.2 安装方式

#### 方式一：原生安装脚本（推荐）

**macOS / Linux / WSL：**

```bash
curl -fsSL https://claude.ai/install.sh | bash
```

**Windows PowerShell：**

```powershell
irm https://claude.ai/install.ps1 | iex
```

**Windows CMD：**

```batch
curl -fsSL https://claude.ai/install.cmd -o install.cmd && install.cmd && del install.cmd
```

> 原生安装会自动在后台更新到最新版本。

#### 方式二：Homebrew（macOS / Linux）

```bash
# 稳定版（约晚一周，跳过有回归的版本）
brew install --cask claude-code

# 最新版
brew install --cask claude-code@latest
```

> Homebrew 安装不会自动更新，需手动运行 `brew upgrade claude-code`。

#### 方式三：WinGet（Windows）

```powershell
winget install Anthropic.ClaudeCode
```

#### 方式四：Linux 包管理器

**Debian/Ubuntu（apt）：**
```bash
echo "deb [signed-by=/usr/share/keyrings/claude-code.gpg] https://packages.anthropic.com/deb stable main" | sudo tee /etc/apt/sources.list.d/claude-code.list
sudo apt update && sudo apt install claude-code
```

**Fedora/RHEL（dnf）：**
```bash
sudo dnf config-manager --add-repo https://packages.anthropic.com/rpm
sudo dnf install claude-code
```

**Alpine（apk）：**
```bash
echo "https://packages.anthropic.com/alpine" | sudo tee -a /etc/apk/repositories
sudo apk add claude-code
```

#### 方式五：npm（已弃用但可用）

```bash
npm install -g @anthropic-ai/claude-code
```

### 2.3 验证安装

```bash
claude --version
```

### 2.4 认证登录

首次运行 `claude` 时，系统会提示登录。支持的登录方式：

- **Claude 订阅**（Pro / Max / Team / Enterprise）
- **Anthropic Console**（API Key 方式）
- **第三方提供商**（Amazon Bedrock / Google Vertex AI / Microsoft Foundry）

```bash
# 启动交互式会话，首次运行时自动引导登录
claude
```

浏览器会自动打开完成 OAuth 认证。如果需要手动指定认证方式：

```bash
# 使用 API Key 登录
claude auth login --api-key YOUR_API_KEY
```

## 三、快速入门

### 3.1 第一条指令

进入你的项目目录并启动 Claude Code：

```bash
cd /your/project
claude
```

**第一条指令示例：**

```
解释这个项目的架构和主要模块
```

Claude Code 会自动探索你的代码库，读取关键文件，然后给出项目结构描述。

### 3.2 一个完整的开发循环

```
这个表单提交后页面崩溃了，帮我找到原因并修复
```

Claude Code 会：

1. **收集上下文**：搜索相关文件，读取表单处理代码
2. **定位问题**：分析 Bug 根因（例如空值未校验）
3. **实施修复**：编辑文件，添加校验逻辑
4. **验证结果**：运行测试确认修复有效
5. **提交更改**（如要求）：用描述性信息提交 Commit

工作完成后，按 `Ctrl+D` 或输入 `/exit` 退出。

### 3.3 常用会话命令

| 命令 | 功能 |
|:----|:----|
| `/help` | 显示帮助信息 |
| `/clear` | 清除对话历史 |
| `/init` | 交互式创建 CLAUDE.md |
| `/model` | 切换使用的模型 |
| `/undo` | 撤销上一步文件变更 |
| `/redo` | 重做被撤销的变更 |
| `/compact` | 手动压缩上下文 |
| `/context` | 查看上下文使用情况 |
| `/config` | 打开设置界面 |
| `/doctor` | 诊断配置问题 |
| `/exit` 或 Ctrl+D | 退出 |
| `Esc` | 中断当前操作 |

### 3.4 基础使用技巧

- **`@` 引用文件**：输入 `@` 后按文件名搜索，Claude 会自动读取文件
- **拖拽图片**：将 UI 设计稿拖入终端，Claude 可以理解图片内容
- **`/` 列出命令**：输入 `/` 查看所有可用的命令和 skills
- **Tab 补全**：按 Tab 补全命令、文件名等
- **`↑` 历史命令**：快速重复之前的指令
- **`Shift+Tab`**：循环切换权限模式

## 四、Claude Code 的工作原理

### 4.1 代理循环（Agentic Loop）

Claude Code 的核心是一个**代理循环**，分为三个持续交替的阶段：

```
你的指令 → 收集上下文 → 执行操作 → 验证结果 → （重复直到完成）
```

1. **收集上下文**：搜索文件、读取代码、运行命令获取信息
2. **执行操作**：编辑文件、创建新文件、运行脚本
3. **验证结果**：运行测试、检查输出、确认变更

这三个阶段不断循环，每一步的结果都会反馈到下一步的决策中。你可能提出一个简单的问题只需要阶段 1，而修复一个 Bug 可能需要三个阶段反复多次。

**你始终是循环的一部分**。你可以随时中断（按 `Esc`）或输入更正的指令来引导 Claude。

### 4.2 驱动循环的两大组件

**模型（Model）**：Claude 模型负责理解代码、推理决策。它读取任何语言的代码，理解组件间的连接方式，并判断需要更改什么。

**工具（Tools）**：工具让 Claude 能够执行操作。内置工具分为五类：

| 类别 | 能力 |
|:----|:------|
| **文件操作** | 读取文件、编辑代码、创建/重命名/删除文件 |
| **搜索** | 按模式查找文件、按正则搜索内容、探索代码库 |
| **执行** | 运行 Shell 命令、启动服务器、运行测试、Git 操作 |
| **Web** | 搜索网络、获取文档、查询错误信息 |
| **代码智能** | 查看类型错误、跳转到定义、查找引用 |

Claude 根据你的指令和在执行过程中获得的信息自主选择使用哪些工具。

### 4.3 会话管理

Claude Code 将会话保存为纯文本 JSONL 文件（位于 `~/.claude/projects/`），支持：

- **回滚**：每次编辑前对文件做快照，可通过 `/undo` 回退
- **续传**：使用 `claude --continue` 或 `claude --resume` 恢复上次会话
- **分叉**：使用 `--fork-session` 或 `/branch` 创建会话分支，原会话不变
- **命名**：运行中会话自动命名，可在 `/resume` 选择器中查看

### 4.4 上下文管理

上下文窗口保存了对话历史、文件内容、命令输出、CLAUDE.md、自动记忆等。当上下文占满时：

1. Claude Code 自动删除旧的工具输出
2. 如果还不够，会总结对话历史
3. 你的关键指令会保留

**技巧**：
- 持久规则放在 CLAUDE.md 而不是对话中
- 运行 `/context` 查看空间使用情况
- 复杂任务使用子代理隔离上下文

## 五、配置系统详解

### 5.1 配置作用域

Claude Code 采用多层级配置作用域系统：

| 作用域 | 位置 | 影响范围 | 是否共享 |
|:------|:----|:--------|:--------|
| **托管（Managed）** | 服务器下发 / MDM 策略 | 所有用户 | 是（IT 部署） |
| **用户（User）** | `~/.claude/settings.json` | 所有项目 | 否 |
| **项目（Project）** | `.claude/settings.json` | 当前项目协作者 | 是（可提交 Git） |
| **本地（Local）** | `.claude/settings.local.json` | 仅自己当前项目 | 否（自动 gitignore） |

**优先级顺序（高 → 低）**：

托管 > 命令行参数 > 本地 > 项目 > 用户

### 5.2 配置文件格式

```json
{
  "$schema": "https://json.schemastore.org/claude-code-settings.json",
  "permissions": {
    "allow": [
      "Bash(npm run lint)",
      "Bash(npm test *)",
      "Read(~/.zshrc)"
    ],
    "deny": [
      "Bash(curl *)",
      "Read(./.env)",
      "Read(./.env.*)",
      "Read(./secrets/**)"
    ]
  },
  "env": {
    "CLAUDE_CODE_ENABLE_TELEMETRY": "1"
  },
  "model": {
    "default": "sonnet",
    "thinking": {
      "budget_tokens": 16000
    }
  },
  "permissions": {
    "allow": ["Bash(npm test *)"],
    "deny": ["Bash(curl *)", "Read(./.env)"]
  }
}
```

`$schema` 提供 VS Code 自动补全和验证。

### 5.3 常用配置项

| 配置项 | 说明 | 示例值 |
|:------|:----|:------|
| `permissions.allow` | 允许自动执行的命令 | `["Bash(npm test *)"]` |
| `permissions.deny` | 禁止执行的命令 | `["Read(./.env)"]` |
| `model.default` | 默认模型 | `"sonnet"` |
| `advisorModel` | 顾问模型（更强模型辅助） | `"opus"` |
| `autoMemoryEnabled` | 自动记忆开关 | `true` 或 `false` |
| `companyAnnouncements` | 企业公告 | `["欢迎使用 Claude Code"]` |
| `disableAllHooks` | 禁用所有钩子 | `true` |
| `outputStyle` | 输出风格 | `"explanatory"` |

### 5.4 环境变量

Claude Code 支持大量环境变量控制行为，在 `settings.json` 的 `env` 块中设置：

```json
{
  "env": {
    "ANTHROPIC_BASE_URL": "https://custom.gateway.com",
    "API_TIMEOUT_MS": "300000",
    "CLAUDE_CODE_DISABLE_AUTO_MEMORY": "1",
    "CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC": "1",
    "DISABLE_AUTOUPDATER": "1",
    "OTEL_METRICS_EXPORTER": "otlp"
  }
}
```

典型用途：
- `ANTHROPIC_BASE_URL`：自定义 API 网关地址（企业代理、第三方兼容 API）
- `API_TIMEOUT_MS`：API 超时（毫秒）
- `CLAUDE_CODE_DISABLE_AUTO_MEMORY`：禁用自动记忆
- `DISABLE_AUTOUPDATER`：禁用自动更新
- `MAX_THINKING_TOKENS`：限制思维链 token 数

## 六、CLAUDE.md 记忆系统

### 6.1 什么是 CLAUDE.md

`CLAUDE.md` 是项目根目录下的 Markdown 文件，Claude Code 在每次会话开始时自动读取。它为 Claude 提供：

- 项目结构和架构说明
- 编码规范和约定
- 构建和测试命令
- 首选的库和工具
- 常见的开发注意事项

**创建方法**：
- 运行 `/init` 命令，Claude 会扫描项目并自动生成
- 手动创建 `CLAUDE.md` 文件

### 6.2 CLAUDE.md 实战示例

```markdown
# FastAPI 后端项目

## 技术栈
- Python 3.12+，FastAPI，SQLAlchemy 2.0，PostgreSQL
- async/await 模式，Pydantic v2 数据验证

## 项目结构
- `app/` — 应用主代码
  - `api/` — 路由定义（按模块拆分）
  - `models/` — SQLAlchemy ORM 模型
  - `schemas/` — Pydantic 请求/响应模式
  - `services/` — 业务逻辑层
  - `core/` — 配置、数据库会话、工具函数
- `tests/` — pytest 测试（与 app/ 结构镜像）

## 编码规范
- 使用 4 空格缩进
- 所有函数和类必须有类型注解
- 使用 `async def` 定义路由
- 异常使用自定义 HTTPException 子类

## 构建与测试
- 安装依赖：`pip install -r requirements.txt`
- 运行测试：`pytest -v --cov=app`
- 代码检查：`ruff check .`
- 类型检查：`mypy app/`

## 数据库迁移
- 使用 Alembic 管理迁移
- 生成迁移：`alembic revision --autogenerate -m "描述"`
- 应用迁移：`alembic upgrade head`
```

### 6.3 文件位置优先级

Claude Code 按顺序查找以下位置的记忆文件：

1. `CLAUDE.md`（项目根目录，推荐）
2. `.claude/CLAUDE.md`（项目 .claude 目录）
3. `CLAUDE.local.md`（本地覆盖，gitignored）
4. `~/.claude/CLAUDE.md`（全局用户级）

> 根目录 `CLAUDE.md` 优先于 `.claude/CLAUDE.md`。建议团队共享的放根目录，个人偏好放 `~/.claude/CLAUDE.md`。

### 6.4 自动记忆（Auto Memory）

除了手动编写 CLAUDE.md，Claude Code 还会**自动累积学习到的东西**：

- 构建命令（`npm run build`、`make` 等）
- 调试过程中发现的根因
- 你偏好的模式和实践
- 项目特有的技巧和注意事项

自动记忆存储在 `MEMORY.md` 文件中，每次会话自动加载前 200 行或 25KB。

**控制自动记忆**：

```bash
# 在 settings.json 中禁用
{
  "autoMemoryEnabled": false
}

# 或通过环境变量
export CLAUDE_CODE_DISABLE_AUTO_MEMORY=1
```

### 6.5 组织级 CLAUDE.md

企业可以通过托管设置注入组织级的 CLAUDE.md 内容：

```json
{
  "claudeMd": "提交前务必运行 make lint。所有 PR 必须通过代码审查。"
}
```

这在托管设置中配置，用户无法覆盖。

### 6.6 CLAUDE.md 中的 Compact Instructions 段落

为帮助 Claude 在自动压缩上下文时保留关键信息，可以在 CLAUDE.md 中加入此段落：

```markdown
## Compact Instructions

当上下文压缩时，请保留以下信息：
- 当前的 API 变更内容
- 正在开发的功能状态
- 已发现的性能优化点
```

## 七、权限与安全

### 7.1 权限模式

按 `Shift+Tab` 循环切换四种权限模式：

| 模式 | 文件编辑 | Shell 命令 | 适用场景 |
|:----|:--------|:----------|:--------|
| **Default（默认）** | 询问 | 询问 | 日常安全开发 |
| **Auto-accept edits** | 自动允许 | 常用文件系统命令自动允许 | 信任 Claude 的编辑 |
| **Plan mode** | 禁止 | 主要禁止 | 仅分析和规划 |
| **Auto mode** | 自动（背景安全检查） | 自动（背景安全检查） | 高度信任环境 |

### 7.2 精细权限规则

在 `settings.json` 中定义允许和拒绝的规则：

```json
{
  "permissions": {
    "allow": [
      "Bash(npm test *)",
      "Bash(npm run lint)",
      "Bash(git status)",
      "Bash(git diff *)",
      "Read(~/.zshrc)"
    ],
    "deny": [
      "Bash(curl *)",
      "Bash(rm -rf *)",
      "Read(./.env)",
      "Read(./.env.*)",
      "Read(./secrets/**)",
      "Write(./**/.env)"
    ]
  }
}
```

规则采用模式匹配方式，支持通配符 `*`。`allow` 规则允许自动执行，`deny` 规则直接阻止执行。

### 7.3 安全机制

- **快照系统**：每次编辑前自动快照文件，`/undo` 可随时回滚
- **沙箱 Bash 工具**：提供文件系统和网络隔离
- **`.env` 文件保护**：默认禁止读取 `.env` 和 `.env.*` 文件
- **权限提示**：对潜在危险操作（curl、远程连接等）默认提示确认
- **快照只覆盖文件变更**：远程系统操作（API、部署等）无法快照，Claude 会格外谨慎

### 7.4 托管安全策略

企业可以通过托管设置强制执行安全策略：

```json
{
  "allowManagedPermissionRulesOnly": true,
  "permissions": {
    "deny": ["Bash(rm -rf *)", "Bash(curl *)"]
  }
}
```

`allowManagedPermissionRulesOnly` 阻止用户和项目设置自定义权限规则，确保安全策略不可绕过。

## 八、工具系统详解

### 8.1 工具概览

Claude Code 内置了丰富的工具集，按功能分为以下几类：

#### 文件操作

## 九、子代理系统

### 9.1 子代理概念

子代理（Subagent）是 Claude Code 中独立运行的 AI 会话，用于：

- **隔离上下文**：子代理有自己独立的上下文窗口，不影响主会话
- **并行执行**：多个子代理可以同时处理不同任务
- **专业分工**：不同子代理可以使用不同的模型和提示词
- **结果汇总**：完成工作后向主会话返回摘要

### 9.2 创建和使用子代理

在对话中通过指令创建子代理：

```
创建一个子代理来分析数据库架构，找出性能瓶颈
```

Claude 会自动分配任务、监控进度，并汇总结果。你也可以在 `settings.json` 中预定义子代理：

```json
{
  "agent": {
    "code-reviewer": {
      "description": "审查代码质量与安全性",
      "model": "opus",
      "prompt": "你是一位严格的高级代码审查员，专注于发现安全漏洞和性能问题。"
    },
    "docs-writer": {
      "description": "编写 API 文档",
      "model": "sonnet",
      "prompt": "你是一位技术文档作者，编写清晰、全面的 API 文档。"
    }
  }
}
```

调用预定义的子代理：

```
@code-reviewer 审查本次提交的所有变更
```

### 9.3 子代理指令集

创建子代理时支持的配置选项：

| 选项 | 说明 |
|:----|:----|
| `description` | 子代理描述，用于选择 |
| `model` | 使用的模型（如 `opus`、`sonnet`） |
| `prompt` | 自定义系统提示词 |
| `maxSteps` | 最大步骤数 |
| `tools` | 允许使用的工具列表 |

### 9.4 Agent Teams（代理团队）

Claude Code 支持协调多个 Claude Code 实例组成团队工作：

- 一个主代理分配任务
- 多个工作代理并行执行
- 代理间消息传递
- 中央化管理

在终端中运行 `claude agents` 或 `/background` 启动背景代理和代理视图。

### 9.5 动态工作流（Dynamic Workflows）

对于大规模任务（如代码审计、大规模迁移），可以使用动态工作流：

- Claude 编写协调脚本
- 脚本可以重复运行
- 适用于代码库审计、大规模迁移、交叉验证研究

## 十一、MCP 服务器集成

### 11.1 什么是 MCP

MCP（Model Context Protocol）是 Anthropic 发起的开放标准，用于连接 AI 工具和外部数据源。通过 MCP，Claude Code 可以：

- 读取 Google Drive 的设计文档
- 更新 Jira 的工单状态
- 从 Slack 拉取消息
- 连接数据库执行查询
- 使用自定义的内部工具

MCP 生态已有超过 300 个可用服务器。

### 11.2 配置 MCP 服务器

MCP 配置存储在 `~/.claude.json`（用户范围）或 `.mcp.json`（项目范围）。

**本地 MCP 服务器**：

```json
{
  "mcpServers": {
    "my-database": {
      "type": "local",
      "command": ["npx", "-y", "@my-org/db-mcp"],
      "env": {
        "DB_HOST": "localhost",
        "DB_PORT": "5432"
      }
    }
  }
}
```

**远程 MCP 服务器**：

```json
{
  "mcpServers": {
    "sentry": {
      "type": "remote",
      "url": "https://mcp.sentry.dev/mcp",
      "oauth": {}
    }
  }
}
```

### 11.3 常用 MCP 服务器

| 服务器 | 用途 | 配置方式 |
|:------|:----|:--------|
| **Sentry** | 查询错误和性能数据 | 远程 + OAuth |
| **GitHub** | 管理 Issue、PR、代码审查 | 远程 |
| **Context7** | 搜索技术文档 | 远程 |
| **Vercel Grep** | 搜索 GitHub 代码示例 | 远程 |
| **Chrome DevTools** | 控制浏览器、调试网页 | 本地 |
| **智谱 Vision** | 图片解析 | 本地 |
| **智谱 Search** | 联网搜索 | 本地 |
| **智谱 Reader** | 网页内容读取 | 本地 |

### 11.4 MCP 身份认证

远程 MCP 服务器支持 OAuth 认证：

```bash
# 手动触发 OAuth 流程
claude mcp auth sentry

# 列出所有 MCP 服务器及认证状态
claude mcp list

# 登出
claude mcp logout sentry
```

### 11.5 MCP 工具搜索

当配置了大量 MCP 服务器（成百上千个工具）时，Claude Code 使用**工具搜索**机制——工具的完整定义按需加载，只有工具名称占用上下文。当 Claude 需要使用某个工具时，才会加载其完整定义。这使得 MCP 可以扩展到数千个工具而不会撑爆上下文。

## 十二、钩子系统（Hooks）

### 12.1 什么是 Hooks

Hooks 是在 Claude Code 特定事件前后自动执行的 Shell 脚本或 HTTP 请求。用于自动化工作流：

- 每次编辑后自动格式化代码
- 每次执行命令前验证安全性
- 任务完成时发送通知
- 在特定目录变更时触发额外处理

### 12.2 配置 Hooks

在 `.claude/settings.json` 中配置：

```json
{
  "hooks": {
    "PostToolUse": {
      "Matcher": {
        "tool": "Edit"
      },
      "Shell": "npx prettier --write $FILE"
    },
    "PreToolUse": {
      "Matcher": {
        "tool": "Bash",
        "command": "rm -rf *"
      },
      "Shell": "echo '高危命令已阻止' && exit 1"
    },
    "TaskComplete": {
      "Shell": "terminal-notifier -title 'Claude Code' -message '任务完成！'"
    }
  }
}
```

### 12.3 钩子事件类型

| 事件 | 触发时机 | 典型用途 |
|:----|:--------|:--------|
| `PreToolUse` | 工具调用前 | 命令验证、阻止危险操作 |
| `PostToolUse` | 工具调用后 | 代码格式化、日志记录 |
| `TaskComplete` | 任务完成 | 发送通知、CI 触发 |
| `ConfigChange` | 配置变更 | 重新加载设置 |

### 12.4 HTTP Hooks

除了 Shell 脚本，Hooks 也可以发送 HTTP 请求：

```json
{
  "hooks": {
    "TaskComplete": {
      "URL": "https://hooks.slack.com/services/xxx/yyy/zzz",
      "Method": "POST",
      "Body": {
        "text": "Claude Code 任务 '$TASK_DESCRIPTION' 已完成"
      }
    }
  }
}
```

## 十五、实战工作流程

### 15.1 代码库探索

刚接手一个新项目：

```
先不要修改任何代码。帮我了解这个项目的整体架构：
1. 项目的技术栈和框架
2. 主要模块和目录结构
3. 数据流向（从入口到数据库）
4. 核心业务逻辑在哪里
```

### 15.2 修复 Bug

```
修复用户登录后看到空白页面的问题。
相关的代码在 src/auth/ 目录下。
```

Claude Code 的修复流程：
1. 读取相关代码理解登录流程
2. 通过错误信息或代码分析定位根因
3. 实施修复
4. 运行测试验证

### 15.3 实现新功能

先规划再实施：

**Phase 1 — 规划（Plan 模式）**：
1. 按 `Shift+Tab` 两次切换到 Plan 模式
2. 描述需求："我们需要给博客添加阅读量统计"
3. 审查 Claude 的计划，提出反馈
4. 确认方案

**Phase 2 — 实施**：
1. 按 `Shift+Tab` 切回 Default 模式
2. "按照计划开始实施"

### 15.4 代码重构

```
将项目中所有的 var 声明替换为 const 或 let。
先搜索所有 var 的位置，批量替换，然后运行测试确保没问题。
```

### 15.5 生成 Commit 和 PR

```bash
claude "提交我的变更，用描述性的提交信息，然后创建 PR"
```

Claude 会自动：
1. 运行 `git diff` 查看变更
2. 生成合适的提交信息
3. 创建分支并提交
4. 使用 `gh` CLI 创建 PR

### 15.6 分析日志输出

```bash
tail -200 app.log | claude -p "分析这个日志，发现异常模式并给出建议"
```

或通过管道操作：

```bash
git diff main --name-only | claude -p "审查这些变更文件的安全性"
```

### 15.7 大型代码库配置

对于单体仓库（Monorepo）或超大型项目：

```json
{
  "claudeMdExcludes": ["**/vendor/**/CLAUDE.md"],
  "permissions": {
    "allow": ["Read(packages/web/**)"]
  }
}
```

使用分层 CLAUDE.md（每个子包独立）、稀疏工作树、代码智能插件来保持聚焦。

## 十六、最佳实践

### 16.1 编写清晰指令

- **具体明确**：不要说"修 Bug"，要说"用户输入错误密码后跳转到空白页，检查 auth/login.ts 的重定向逻辑"
- **提供上下文**：引用相关文件（用 `@`），贴出错误消息
- **分步说明**：复杂任务拆解为步骤
- **提供验证标准**："实现后运行 `npm test` 确保通过"

### 16.2 高效协作技巧

- **先探索再实施**：用 Plan 模式规划复杂任务
- **从简单开始**：先小范围验证，再扩大到整个项目
- **迭代改进**：第一次结果不满意，补充说明继续调整
- **使用子代理**：大数据量任务用子代理隔离上下文
- **善用 `/undo`**：结果不符合预期及时回退

### 16.3 项目配置最佳实践

- **CLAUDE.md 提交到 Git**：让团队共享项目知识
- **定期运行 `/init`**：项目结构变化后更新 CLAUDE.md
- **自动记忆**：保持开启，Claude 会在工作中自动学习
- **自定义命令**：常用工作流配置为 `/command`
- **权限精细化**：常用安全命令加 `allow`，危险命令加 `deny`

### 16.4 成本管理

- **使用 Sonnet 作为默认模型**，Opus 仅在复杂任务时切换
- **控制 extended thinking**：只在需要深度推理时开启
- **定期运行 `/context`**：检查上下文使用情况
- **用 `/compact` 手动压缩**：在上下文增长过大时主动压缩
- **善用预处理 Hooks**：在发送前过滤掉不必要的输出

### 16.5 安全最佳实践

- 保持 Default 权限模式
- 在 settings.json 中 `deny` 危险命令
- 关键任务使用 Plan 模式先行审查
- 手动检查 Claude 创建的 PR 后再合并
- 企业环境使用托管设置强制执行安全策略

### 16.6 诊断工具

```bash

claude doctor

# 查看上下文中加载了什么
/context

# 查看已加载的钩子
/hooks

# 查看 MCP 服务器状态
/mcp

# 检查各功能对 plan 限制的消耗
/usage
```

## 十八、常见问题

### 18.1 安装问题

**Q: 提示 `claude: command not found`**

A: 原生安装脚本会将二进制放在 `~/.local/bin/`，确认该目录在 PATH 中。或重新登录终端使 PATH 生效。

**Q: Windows 下 PowerShell 执行报错**

A: 确保使用正确的 Shell：
- PowerShell 用 `irm` 命令
- CMD 用 `curl` 命令
- 查看提示符：`PS C:\>` 是 PowerShell，`C:\>` 是 CMD

### 18.2 认证问题

**Q: 登录时浏览器无法打开**

A: 使用 `claude auth login` 手动登录，或将认证码粘贴到终端。

### 18.3 配置问题

**Q: settings.json 不生效**

A: 检查 JSON 格式是否正确。运行 `claude doctor` 查看配置加载状态。

**Q: 权限规则没按预期工作**

A: 规则优先级：托管 > 本地 > 项目 > 用户。确认规则没有被更高优先级覆盖。运行 `/context` 查看实际生效的配置。

### 18.4 性能问题

**Q: Claude Code 响应很慢**

A: 检查网络连接。尝试切换到 Sonnet 或 Haiku 模型。使用 `/fast` 开启快速模式。检查 `API_TIMEOUT_MS` 是否过短。

**Q: 上下文很快用完**

A: 使用 `/compact` 手动压缩。将持久指令放在 CLAUDE.md 中。使用子代理隔离大数据量任务。

### 18.5 文件问题

**Q: Claude 找不到某些文件**

A: 文件可能被 `.gitignore` 排除。检查文件路径是否正确。使用 Glob 工具搜索确认文件存在。
